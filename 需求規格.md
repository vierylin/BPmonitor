# 【AI Agent 開發需求規格書】智能血壓分析與臨床戰情室 Dashboard

## 一、 專案概述

開發一款輕量級的網頁應用程式（Web App），專供臨床醫師在看診或查房時快速瀏覽患者血壓趨勢。系統需支援手動輸入與 Excel 批次上傳，並能自動繪製視覺化圖表（折線、柱狀、SPC），最後透過串接 LLM 生成精準的臨床洞察與建議。

## 二、 建議技術棧 (可請 Agent 依此配置)

* **前端**：HTML/JavaScript + Tailwind CSS (確保 UI 乾淨、高對比度，適合醫療環境) + 視覺化套件 (推薦使用 ECharts 或 Chart.js，對 SPC 支援度佳)。
* **後端/資料處理**：Python (FastAPI) 搭配 Pandas 進行資料清洗與數值統計。
* **雲端/部署**：Firebase 進行前端託管，或部署於 Google Cloud Platform (GCP)。

## 三、 核心功能規格 (Core Features)

### 1. 數據輸入模組 (Data Input)

* **手動單筆輸入**：提供直覺的表單，包含日期 (Date)、時間 (Time)、收縮壓 (SBP)、舒張壓 (DBP)、心率/脈搏 (HR)。並自動計算平均動脈壓 (MAP = (SBP + 2*DBP)/3)。
* **Excel 批次上傳**：
* 支援 `.xlsx` 或 `.csv` 格式。
* **防呆與容錯機制**：自動識別欄位（允許中英文欄位名：日期、時間、收縮壓、舒張壓、脈搏），若格式錯誤需跳出具體提示。
* 自動過濾空白列與極端異常值（如 SBP > 300 或 < 40）。



### 2. 進階臨床數據分析 (主流軟體擴充功能)

* **時間區段切換**：支援「近 7 天」、「近 1 個月」、「近 3 個月」快速切換。
* **日夜間血壓對比 (Morning vs. Evening)**：自動將數據分為早晨（如 06:00-10:00）與夜間（如 18:00-22:00），這對於判斷夜間高血壓 (Non-dipper) 具備重要臨床意義。
* **分級警戒色碼**：依據高血壓治療指引（如：正常、高血壓前期、Stage 1、Stage 2），在圖表與數據表上使用紅、橘、綠等顏色標註，讓醫師一眼看出異常。

### 3. 圖表視覺化模組 (Data Visualization)

* **綜合折線圖 (Line Chart)**：顯示 SBP, DBP, MAP 的長期趨勢，背景區塊需標示出標準目標值區間（例如 SBP 130 以下）。
* **分佈柱狀圖 (Bar Chart)**：顯示各血壓分級的次數佔比，快速了解患者血壓達標率 (Time in Target Range)。
* **SPC 統計製程管制圖 (Statistical Process Control)**：
* 繪製 X-bar (平均值) 與 R (全距) 或 S (標準差) 管制圖。
* 自動計算並標示中心線 (Center Line)、管制上限 (UCL)、管制下限 (LCL)。
* 自動圈出違反 SPC 規則的異常點（如連續 7 點在同一側、超出 3 個標準差等），協助醫師抓出隱藏的病情變化。



### 4. LLM 智能分析與建議模組

* **觸發機制**：資料整理完畢後，後端透過 API (如 OpenAI 或 Gemini) 傳送「精簡後的數據摘要與統計特徵」給 LLM，避免傳送過多原始資料導致 Token 浪費。
* **System Prompt 要求**：LLM 需扮演「資深內科/腎臟科主治醫師」的角色，語氣需專業、客觀、條理分明。
* **輸出內容需包含**：
1. **趨勢總結**：血壓波動型態（如：早晨血壓偏高、近期起伏變大）。
2. **可能風險提示**：例如潛在的體液容積過載 (Volume overload)、或是藥物劑量可能需要調整的訊號。
3. **臨床建議**：條列式給予用藥調整方向或生活型態衛教建議。



### 5. 臨床戰情室輸出 (Dashboard UI)

* **版面配置 (Layout)**：
* **頂部**：患者基本數據摘要 (平均 SBP/DBP、達標率)、LLM 的「一句話核心結論」。
* **中部**：互動式圖表 (可縮放、點擊查看具體數值)。
* **底部/側邊欄**：LLM 完整分析報告與建議。


* **匯出功能 (Export)**：一鍵將當前 Dashboard 匯出為乾淨的 PDF 報表，方便列印給患者衛教，或作為電子病歷 (EMR) 的附件。
